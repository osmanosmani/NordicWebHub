# Project Review Checklist

Use this checklist before a technical review, portfolio screenshot session, or GitHub release.

## Build and Test

- [ ] Backend restore succeeds.
- [ ] Backend build succeeds.
- [ ] Backend tests pass.
- [ ] Frontend install succeeds.
- [ ] Frontend build succeeds.
- [ ] Frontend lint succeeds.

## Manual App Review

- [ ] Public landing page loads.
- [ ] Pricing page loads.
- [ ] Login works for Admin.
- [ ] Login works for Customer.
- [ ] Logout works.
- [ ] Refresh keeps the authenticated user.
- [ ] Unauthorized role access is blocked.

## Admin Area

- [ ] Admin dashboard loads.
- [ ] Companies page works.
- [ ] Service Packages page works.
- [ ] Project Requests page works.
- [ ] Projects page works.
- [ ] Support Tickets page works.
- [ ] Service Orders page works.
- [ ] Maintenance Logs page works.
- [ ] SEO Reports page works.
- [ ] Website Check page works.

## Customer Area

- [ ] Customer dashboard loads.
- [ ] My Company page works.
- [ ] Project Requests page works.
- [ ] Projects page works.
- [ ] Support Tickets page works.
- [ ] Service Orders page works.
- [ ] Maintenance Logs page works.
- [ ] SEO Reports page works.
- [ ] Hosting Status page works.
- [ ] AI Service Assistant works.

## UI and Responsiveness

- [ ] Navigation works on desktop.
- [ ] Navigation works on mobile.
- [ ] Tables are readable or scrollable on small screens.
- [ ] Forms have clear labels.
- [ ] Loading states are visible.
- [ ] Empty states are clear.
- [ ] Error states are clear.
- [ ] No obvious horizontal overflow.

## Documentation

- [ ] README is up to date.
- [ ] Setup instructions are accurate.
- [ ] Screenshot checklist exists.
- [ ] Architecture docs exist.
- [ ] Security checklist exists.
- [ ] Deployment notes exist.
- [ ] Future improvements are honest and realistic.

## GitHub Readiness

- [ ] No secrets are committed.
- [ ] `.gitignore` is correct.
- [ ] Local config files are ignored.
- [ ] Remote URL is correct.
- [ ] Working tree is clean before push.
